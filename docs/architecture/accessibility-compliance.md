### Keyboard Navigation

All interactive elements are reachable via Tab key:
1. Skip-to-main link (visible on first Tab)
2. Sidebar navigation links
3. Page content (headings, tables, forms, buttons)
4. DataTable: sortable headers (click/Enter), interactive rows (Enter/Space)
5. Forms: all inputs, selects, textareas, buttons
6. Mobile: hamburger menu (Escape to close)

### Screen Reader Support

- **Landmarks:** `<nav aria-label="Main navigation">`, `<main id="app-main">`, `<header role="banner">`
- **Tables:** `<caption>` (visually hidden), `scope="col"`, `aria-sort`
- **Forms:** `<fieldset>`/`<legend>`, `aria-required`, `aria-invalid`, `aria-describedby`
- **Status:** `role="status"` on StatusBadge, `aria-live="polite"` for dynamic updates
- **Tabs:** `role="tablist"`, `role="tab"`, `role="tabpanel"`, `aria-selected`

### Motion Sensitivity

`prefers-reduced-motion: reduce` media query disables all animations and transitions.

## Testing Procedures

### Automated
- ESLint with `eslint-plugin-jsx-a11y` (recommended ruleset)
- Lighthouse accessibility audit (target: 95+)

### Manual
1. **Keyboard-only test:** Complete every workflow without a mouse
2. **Screen reader test:** VoiceOver (macOS) through all pages
3. **Contrast check:** WebAIM contrast checker on all text/background combinations
4. **Zoom test:** 200% browser zoom — no content loss or overlap
5. **Reduced motion test:** Enable "Reduce motion" in System Preferences
